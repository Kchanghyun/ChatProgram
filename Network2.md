# 네트워크 모델의 종류
1. TCP/IP 모델
2. **OSI 7계층** : 1984년 네트워크 통신을 체계적으로 다루는 ISO에서 표준으로 지정한 모델, 데이터를 주고받을 때 데이터 자체의 흐름을 각 구간별로 나눠 놓은 것

|계층|단계|종류|
|:--:|:--:|:--:|
|7계층|응용|**HTTP**, SMTP....|
|6계층|표현|SMB....|
|5계층|세션|NetBios|
|4계층|전송|**TCP, UDP**...|
|3계층|네트워크|**IP, ICMP, ARP**...|
|2계층|데이터 링크|**이더넷**...|
|1계층|물리|전선, 전파, 광섬유, 리피터...|

#### 공통점
1. 계층적 네트워크 모델
2. 계층 간 역할 정의

#### 차이점
1. 계층의 수 차이
2. OSI는 역할 기반(논리), TCP/IP는 프로토콜 기반(실무)
3. OSI는 통신 전반에 대한 표준
4. TCP/IP는 데이터 전송기술 특화

## 패킷
네트워크 상에서 전달되는 데이터를 통칭하는 말로 네트워크에서 전달하는 데이터의 형식화된 블록이다. 패킷은 제어 정보와 사용자 데이터로 이루어지며 사용자 데이터는 페이로드라고 한다.

|--|--|--|
|--|--|--|
|헤더|페이로드|풋터|
|--|--|--|

|--|--|--|--|
|--|--|--|--|
|Ethernet|IPv4|TCP|데이터|
|--|--|--|--|

#### 보낼 때(캡슐화)
1. 페이로드 : HTTP\
헤더 : TCP

2. 페이로드 : TCP + HTTP\
헤더 : IPv4

3. 페이로드 : IPv4 + TCP + HTTP\
헤더 : Ethernet

#### 받을 때(디캡슐화)
1. 페이로드 : IPv4 + TCP + 데이터\
헤더 : Ethernet

2. 페이로드 : TCP + 데이터\
헤더 : IPv4

3. 페이로드 : 데이터\
헤더 : TCP

### PDU(계층 별 이름)
4계층의 PDU(TCP + 데이터) = 세그먼트\
3계층의 PDU(IPv4 + TCP + 데이터) = 패킷(위의 패킷과는 다른 뜻, 일반적으로 패킷이라 하면 이 뜻보다 위의 전달되는 데이터를 통칭하는 용어인 패킷의 뜻으로 사용된다. )\
2계층의 PDU(Ethernet + IPv4 + TCP + 데이터) = 프레임
